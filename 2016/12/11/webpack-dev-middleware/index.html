webpack-dev-middleware

webpack-dev-middleware作用于级联的中间件堆栈，当你已经有一个Node.js的服务或者你希望当前服务在你的完全掌控下是极其有用的。
该中间件将会导致webpack在内存中编译文件。当一个编译正在运行时，它会延迟当前请求直到编译完成。
该中间件是为高级用户使用的，相对比webpack-dev-server更易用
从npm安装依赖：
npm install express webpack-dev-middleware --save-dev

参数

devServer.clientLogLevel [string]

在inline模式下，你的DevTools控制台会打印一些信息，比如在重新加载之前、发生错误或者热更新开启时，这可能会非常的冗余。
当然你可以使用以下参数来选择所需要的通知类别：
clientLogLevel: "none"
可用的值有none,error,warning或info（默认）
需要知道的时控制台会始终显示bundle的错误及警告信息。

devServer.compress [boolean]

在所有服务中开启gzip压缩
compress: true

devServer.contentBase [boolean] [string] [array]

用来告诉服务器所提供服务的内容来源。此参数仅在你想使用静态文件服务的情况下时必须的。devServer.publicPath将会用于优先去确定bundles从哪里加载。
在默认情况下它将会加载你的默认工作目录，但你可以通过修改此项参数去更改服务目录：
contentBase: path.join(__dirname, "public")
需要注意的是，这条命令将会使用绝对路径。
当然它也可以同时为几个目录提供服务：
contentBase: [path.join(__dirname, "public"), path.join(__dirname, "assets")]
禁用contentBase:
contentBase: false

devServer.filename [string]

该参数允许你在lazy模式下减少编译次数。在lazy模式默认情况下，每一次的请求都会进行一次新的编译过程，使用filename，将允许你只有在请求特定文件的时候进行编译。

只有在/bundle.js被请求时，才会编译当前bundle

filename 在不使用lazy模式的情况下是无效的。

devServer.headers [object]

给所有请求添加headers

devServer.historyApiFallback [boolean] [object]

在使用HTML5 History Api时，index.html将会替代404页面，通过以下来启用此功能：
historyApiFallback: true

devServer.host - CLI only（仅用于CLI）[string]

指定一个主机host来使用，默认情况下使用localhost，如果你想要使用一个外部服务，通过以下来指明：
host: "0.0.0.0"

devServer.hot [boolean]

启用webpack的热更新特性：
hot: true

在设置此项时需要设置额外步骤。（以我的经验和相关的文档，貌似在此处需要设置其他步骤——并不是在命令行，它仅仅是一个标志？）
devServer.https boolean object

在默认情况下dev-server会通过HTTP提供服务，它也可配置为HTTP/2协议的HTTPS提供服务。
https: true

devServer.noInfo [boolean]

当noInfo启用时，譬如webpack的bundle信息将会在项目开始和每次保存后将会隐藏，但是errors及warnings信息将会始终显示。

当前打包后的文件在浏览器中将会在此路径下可用。

devServer.quiet [boolean]

启用quiet后，除了初始化信息意外将会没有任何内容在控制台输出，这也就意味着errors和warnings信息也是不可见的。
