css中百分数计算参考对象
  • padding 百分数总是跟包含块的宽度有关，高度不考虑在内
  • margin 百分数的取值规则与padding内边距时一样的，与包含块的宽度有关。（Tap:外编剧在display设置为inline时，顶边和底边的外边距时不会显示的）
  • width&height 百分数值是根据包含块的宽度来计算的（处理元素的高度也是按照这个宽度来计算）
    浏览器对box-sizing的支持
  • 现代浏览器都能支持box-sizing属性，基本支持content-box和border-box
  • IE家族>=8， 其中在IE8中，若设置了min/max-width/height属性值，那么box-sizing的值将会被忽略
  • Safari 6.0.x在diaplay:table显示模式中不会使用box-sizing属性
  • IE9中，当设置position:absolute/fixed,overflow:auto/overflow-y:scroll时，将会从元素的宽度减去滚动条的宽度
  • 安卓浏览器在HTML的select元素上并不会计算精确的尺寸（width和height）
    行内元素（inline）
  • 使用行内元素显示inline时，浏览器会忽略当前元素的某些属性，如width、height、margin-top、margin-bottom，经测试，margin-left和margin-right还是有效的
  • 若使用行内-块级元素inline-block，那么width、height、margin的属性军可以应用到盒子上
    盒类型（display）
  • run-in:其盒子类型取决于周围元素，目前测试主流浏览器已经舍弃了该属性值
    • 如果插入元素包含一个display属性为block的元素，那么插入元素就是块级元素。
    • 如果插入元素的相邻兄弟元素时块级元素，那么插入元素就是兄弟元素中的第一个行内元素。
    • 其他情况下，插入元素均作为块级元素来对待。
      设置float属性
  • 在JavaScript中设置元素的float属性需要使用cssFloat，即element.style.cssFloat = left/right/none
    定位类型（position）
  • static： 元素为普通布局，为默认值
  • relative：元素位置相对于普通位置而定
  • absolute：元素位置相对于position不为static的第一位祖先元素来定位

  • fixed：元素相对于浏览器窗口来定位

    css table布局与html中table元素的对应关系

  • table 类似<table>元素

  • inline-table 类似<table>元素，但是创建一个行内元素
  • table-caption 类似<caption>元素
  • table-column 类似<col>元素
  • table-column-group 类似<colgroup>元素
  • table-header-group 类似<thead>元素
  • table-row-group 类似<tbody>元素
  • table-footer-group 类似<tfoot>元素
  • table-row 类似tr元素
  • table-cell 类似td元素

text-justify属性值的适用情况
  • auto 浏览器选择对其规则，这是最简单的方法，不过，不同浏览器之间的呈现方式会有微小的差别
  • none 禁用文本对齐
  • inter-word 空白分布在单词之间，适用于英语等词间有间隔的语言
  • inter-ideograph 空白分布在单词、表意字之间，且文本两端对齐，适用于汉语、日文和韩文等语言
  • inter-cluster 空白分布在单词、字形集的边界，适用于泰文等无词间空格的语言
  • distribute 空白分布在单词、字形集的边界，但连续文本或者草体除外
  • kashida 通过拉长选定字符调整对齐方式（仅适用于草体）
