Object类型

在使用对象字面量表示法来定义对象的时候，在最后一个属性后面添加逗号，会在**IE7及更早版本和Opera中导致错误`。
其中：

在通过对象字面量定义对象时，实际上不会调用Object构造函数。

对象字面量也是向函数传递大量可选参数的首选方式。
从功能上来看，以下两种访问对象属性的方式没有任何区别：

// 使用Array构造函数创建数组：
var colors = new Array(); // 创建一个空数组
var colors = new Array(20); // 创建一个length为20的数组
var colors = new Array(“Greg”); // 创建一个包含 1 项， 即字符串”Greg”的数组

// 使用数组字面量表示法：
var colors = [“red”, “blue”, “green”];
var names = []; // 创建一个空数组

// !!!不建议的操作
var values = [1, 2, ]; // 这样会创建一个包含 2 或 3 项的数组
var options = [ , , , , , ]; // 这样会创建一个包含 5 或 6 项的数组

// 以下是等价的，在直接传入构造函数的时候，也会在后台调用Date.parse()
var someDate = new Date(Date.parse(“May 25, 2017”));
var someDate = new Date(“May 25, 2017”);

// 以下也是等价的
var y2k = new Date(Date.UTC(2017, 0));
var y2k = new Date(2017, 0);

// 以下两种定义方式是等价的
var pattern1 = /[bc]at/i;
var pattern2 = new RegExp(“[bc]at”, “i”);

return num1 + num2;
}

var sum = function(num1, num2) {

var sum = new Function(“num1”, “num2”, “return num1 + num2”); //不推荐

“use strict”

console.log(this);

return sum.apply(this, arguments);
